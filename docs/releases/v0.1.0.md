# Release v0.1.0

`v0.1.0` is the first public release of CogniWeave Memory under the Python package name `cogni-mem`.

## Highlights

- Published to PyPI as `cogni-mem==0.1.0`
- Based on the internal milestone `范式+记忆系统&RAG重构 v0.2`
- Keeps the framework core behavior aligned with that source milestone
- Adds standard Python packaging via `pyproject.toml`
- Splits optional dependencies into extras instead of forcing all integrations into the base install
- Adds bilingual release-oriented README and architecture notes
- Ships under the MIT License

## Installation

```bash
pip install cogni-mem==0.1.0
```

Optional extras:

```bash
pip install "cogni-mem[minimax]==0.1.0"
pip install "cogni-mem[graph]==0.1.0"
```

Install from the GitHub tag:

```bash
pip install "git+https://github.com/luckly06/cogniweave-memory.git@v0.1.0"
```

## Scope

This release is intentionally packaging-focused. It does not introduce framework behavior changes beyond the metadata, dependency layering, and documentation work required for public distribution.

## Runtime Model

- Default local runtime:
  key memory uses JSON, metadata uses SQLite, vector memory uses local Qdrant path mode
- Optional remote Qdrant:
  enable with `COGNIWEAVE_ENABLE_QDRANT=true`
- Optional Neo4j graph integration:
  install the `graph` extra and enable `COGNIWEAVE_ENABLE_NEO4J=true`
